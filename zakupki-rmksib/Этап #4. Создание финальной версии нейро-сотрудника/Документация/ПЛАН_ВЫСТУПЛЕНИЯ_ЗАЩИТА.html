3.2. Система оценки рисков

Текст:

"Реализована система количественной оценки рисков по 10-балльной шкале:"

Параметры оценки:

  1. Редкость/нишевость товара (вес 0.2)
  2. Риски санкций (вес 0.3)
  3. Превышение бюджета (вес 0.3)
  4. Другие факторы (вес 0.2)

Интегральная оценка риска:

  • Рассчитывается по формуле с весовыми коэффициентами
  • Интерпретация: ≤4.0 — участвовать, 4.1-6.0 — участвовать с осторожностью, >6.0 — не участвовать

Пример результата:

  • Редкость товара: 2/10 — массовый товар
  • Риски санкций: 6/10 — умеренные риски
  • Превышение бюджета: 0/10 — бюджет в пределах нормы
  • Другие факторы: 5/10 — стандартные условия
  • Интегральная оценка: 3.4/10 → Рекомендация: Участвовать
Визуал: Пример структурированного ответа нейро-сотрудника

3.3. Эксперименты и оптимизация

Текст:

"Проведена серия экспериментов для оптимизации работы нейро-сотрудника:"

  • Среднее время выполнения: 14.2 секунды
  • Оптимальные параметры: sonar, temperature=0.2, max_tokens=2000
  • Для сложных задач рекомендуется sonar-pro (время 24 сек, но более детальный анализ)

ПОДГОТОВКА К ВОПРОСАМ

Вопрос 1: "Почему выбрали Perplexity, а не OpenAI GPT?"

Ответ:
  • Perplexity имеет встроенный онлайн-поиск, что критично для актуальной информации о санкциях и ценах
  • Более доступная стоимость использования
  • Хорошая поддержка русского языка
  • В будущем планируется интеграция с другими моделями для сравнения

Вопрос 2: "Как обеспечивается точность оценок нейро-сотрудника?"

Ответ:
  • Использование структурированных промптов с четкими требованиями
  • Количественная оценка по 10-балльной шкале с обоснованием
  • Интегральная оценка с весовыми коэффициентами
  • Проведение экспериментов для оптимизации параметров
  • В будущем планируется обучение на исторических данных для улучшения точности

Вопрос 3: "Какие ограничения у системы?"

Ответ:
  • Зависимость от внешнего API (требуется стабильный интернет)
  • Ограничения по количеству запросов (rate limits)
  • Обработка больших документов ограничена 10,000 символов
  • Требуется валидация результатов человеком для критичных решений

Вопрос 4: "Как система обрабатывает ошибки?"

Ответ:
  • Реализована обработка ошибок API (401, 400, таймауты)
  • Автоматические повторные попытки при сбоях
  • Логирование всех запросов и ответов
  • Уведомления администраторам об ошибках

Вопрос 5: "Планируется ли коммерческое использование?"

Ответ:
  • Система разработана для внутреннего использования компанией РМКСИБ
  • В перспективе возможно масштабирование для других компаний
  • Требуется доработка для коммерческого использования (безопасность, масштабирование)
