# Инструкция по запуску Django Admin Panel

## Проблема: Страница не найдена

Если вы видите ошибку "Страница не найдена" при открытии http://localhost:8000/admin/, выполните следующие шаги:

## Шаг 1: Проверка подключения к БД

```bash
python check_db_connection.py
```

Если есть ошибки подключения:
1. Убедитесь, что PostgreSQL запущен
2. Проверьте настройки в `.env` файле в корне проекта
3. Убедитесь, что пользователь БД имеет права доступа

## Шаг 2: Применение миграций

```bash
python manage.py migrate --run-syncdb
```

**Важно:** Если возникает ошибка с кодировкой, убедитесь, что:
- Пароль в `.env` не содержит специальных символов
- PostgreSQL настроен на UTF-8

## Шаг 3: Создание суперпользователя

```bash
python scripts/create_django_admin.py
```

Или используйте стандартную команду:
```bash
python manage.py createsuperuser
```

## Шаг 4: Запуск сервера

```bash
python manage.py runserver
```

Вы должны увидеть:
```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

## Шаг 5: Открытие админ-панели

Откройте в браузере:
```
http://127.0.0.1:8000/admin/
```

Или:
```
http://localhost:8000/admin/
```

## Альтернативный вариант: Запуск на другом порту

Если порт 8000 занят:

```bash
python manage.py runserver 8001
```

Затем откройте:
```
http://localhost:8001/admin/
```

## Решение проблем

### Ошибка: "No such file or directory"
- Убедитесь, что вы находитесь в директории `admin_panel/`
- Проверьте путь: `cd admin_panel`

### Ошибка: "ModuleNotFoundError: No module named 'django'"
- Установите зависимости: `pip install -r ../requirements.txt`

### Ошибка: "Connection refused" или "could not connect"
- Проверьте, что PostgreSQL запущен
- Проверьте настройки в `.env` файле
- Убедитесь, что порт 5432 не заблокирован файрволом

### Ошибка с кодировкой UTF-8
- Убедитесь, что пароль в `.env` не содержит специальных символов
- Проверьте кодировку файла `.env` (должен быть UTF-8)

## Проверка работы сервера

После запуска `python manage.py runserver` вы должны увидеть в терминале:
- Сообщение о запуске сервера
- Логи запросов при обращении к страницам

Если сервер не запускается, проверьте:
1. Нет ли ошибок в выводе команды
2. Не занят ли порт другим процессом
3. Правильно ли настроена БД
